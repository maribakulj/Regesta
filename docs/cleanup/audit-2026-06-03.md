## B. Defects / over-claims introduced this session ‚Äî FIX NOW

These belong to **no** future WP ‚Äî they are regressions/inconsistencies in code we
just wrote (WP-3/WP-5 deliverables) and must be cleaned before resuming WP-4.

- **R1 ‚Äî `norm` drift + eval over-claim.** The OpenLibrary eval uses a `norm` that
  strips punctuation; `frbrise`/`project` `norm` does **not**. So the eval does not
  measure the projection's real strategy, yet its docstring says it does. `norm`
  is also duplicated in 4 places (frbrise, project, eval, spike) ‚Üí drift.
  *Fix:* one shared `norm`; align the eval (or state the difference precisely) and
  correct the docstring.
- **R2 ‚Äî D7 commit policy is inconsistent.** `frbrise` marks the transcribed
  Manifestation title `:asserted`; `project` marks the same kind of transcription
  `:proposed`, so a canonical source certifies to nothing. And `certified-only?`
  export ties entity inclusion to *being referenced by an asserted claim*, so a
  titleless-but-certain Manifestation is dropped.
  *Fix:* one rule for "transcription is certified", applied in both projections;
  include structurally-certain entities in the certified view.
- **R3 ‚Äî the loss report double-counts.** A native field unmapped at import is
  counted again at export (same `:intermarc/*` still on the record) ‚Üí aggregate
  `:total` and `:by-source-field` are inflated ~2√ó vs the ~102 distinct fields.
  Only the per-edge breakdown is clean.
  *Fix:* present per-edge; deduplicate the distinct-field/aggregate figures.
- **R4 ‚Äî `:coerced` over-reports.** It fires whenever a lossy transform is in the
  chain even if the value was unchanged (`lowercase` on already-lowercase).
  *Fix:* fire only when the value actually changed (or document the choice).
- **R5 ‚Äî the OpenLibrary fixture is corrupt.** 84 mojibake lines
  ("Honor√É¬©") + a BOM. `author` is wrong on those rows (harmless to the eval, but
  corrupt committed data).
  *Fix:* regenerate clean (UTF-8, no BOM), or document the corruption + clean on read.
- **R6 ‚Äî honesty pass.** Soften over-claims in test docstrings ("universality
  *proof*" on 2 inline DC records ‚Üí "demonstration on inline DC"); confirm no
  committed doc states IIIF/CLI/real-spokes as done (README/roadmap already honest).
